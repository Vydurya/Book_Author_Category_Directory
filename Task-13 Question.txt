1. Create a REST API server with nestjs and postgresql
2. Also setup entities and relations for the following tables
3. Create the following endpoints

Tables

Books
 - id number autogenerated
 - name text
 - description text
 - author integer
 - category integer
Category
 - id number autogenerated
 - name text
Author
 - id number autogenerated
 - name text

A book can have only one author and category. 
A category can contain multiple books
An author can have multiple books

REST API

Books Creation API
- endpoint name : /createBooks Method POST
- Input body: {name:text, description:text, author: integer, category: integer} // JSON object
- output response  {message : ""} //Respond with SUCCESS or FAILURE

Catogory Creation API
- endpoint name : /createCategory Method POST
- Input body: {name:text} // JSON object
- output response  {message : ""} //Respond with SUCCESS or FAILURE

Author Creation API
- endpoint name : /createAuthor Method POST
- Input body: {name:text} // JSON object
- output response  {message : ""} //Respond with SUCCESS or FAILURE

Get All Books by filter API
- endpoint name : /getAllBooksByFilter Method GET
- Input body: {search:text, authorId: integer, categoryId: integer} // JSON object 
- output response  Respond with Book data, its category details and its author details.

Sample
{search: "HarryPotter", authorId: 1, categoryId: null}
For this request, the result should contain all the books in the table with name HarryPotter and author id 1
